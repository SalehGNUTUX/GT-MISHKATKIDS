## الأوامر

```bash
npm run dev        # خادم تطوير Vite (hot reload)
npm run build      # بناء الإنتاج → dist/ (ملفات ثابتة + PWA service worker). التحقّق الإلزامي بعد أي تعديل.
npm run preview    # معاينة مخرجات dist/
npm run gen:audio  # توليد مقاطع نطق espeak (يتطلّب espeak-ng + lame على جهاز التطوير). بعد أي إضافة محتوى.
npm run gen:voices # دمج تسجيلاتٍ بشريّة مُصدَّرة (voices-src/) في الحزمة (يتطلّب ffmpeg)
npm run gen:custom # مسح public/tts/custom/ وبناء قائمة التسجيلات البديلة اليدوية
npm run gen:piper  # توليد مجموعة صوتٍ عصبيّة (Piper) العربيّة (ar_JO-kareem) للكلمات/الجمل/القصص → voices-bundled.json (وقت التطوير)
npm run gen:quran  # تنزيل نصّ القرآن (alquran.cloud) + تلاوة الحصري آيةً آية (everyayah.com) للفاتحة وجزء عمّ → content/quran.js + public/quran/husary/*.mp3 (إنترنت مرّةً، وقت التطوير)
npm run gen:icons  # استخراج أيقونات Font Awesome Free (solid) المستعمَلة → src/icons.js (بعد إضافة اسمٍ في tools/gen-fa-icons.mjs)
node tools/gen-piper-lang.mjs en|fr   # توليد صوت اللغات الأجنبيّة العصبيّ كاملاً (en_GB-alba / fr_FR-tom): أحرف+كلمات+عبارات+أفعال+قصص → مجموعتا tts-alba/tts-tom (وقت التطوير؛ لازمٌ بعد أيّ إضافة محتوًى لغويّ وإلّا صَمَتَ على لينكس)
```

- لا اختبارات آلية ولا linter. **التحقّق:** `npm run build` (يجب أن ينجح)، و`node --check src/<file>.js` لوحدة، و`node -e "import('./content/library.js').then(m=>console.log(m.default.memories.length))"` لسلامة JSON.
- على قرص ext4 قد يفقد `node_modules/.bin` صلاحية التنفيذ (`vite: Permission denied`) → `chmod +x node_modules/.bin/*`.
- `Date.now()`/`Math.random()` مسموحة في كود التطبيق والأدوات (Node)، لا في سكربتات Workflow.

**مِشكاةُ الطفلِ والآليّ** (Mishkat — The Child & the Robot؛ مجلّد `kidlearn`، الاسمُ الداخليُّ السابق «تلميذي» يبقى في الكود/أسماء الملفّات): تطبيقٌ تعليميٌّ للأطفال (4–10) بفكرة **قلب الأدوار** — الطفل يُعلّم آليّاً فقَدَ ذاكرته (Protégé Effect). شعارُه «الطفلُ يُعلّم، والآليُّ يَستنير» (انظر ذاكرة `project_tilmithi_brand`). في كلّ نصوص الواجهة والتوثيق الشخصيّةُ الودودةُ تُسمّى **«الآليّ»** (استُبدِل الاسمُ السابق «روبو» بالكامل في 2026-06-25؛ مُعرّفاتُ الكود `robo`/`robo-companion`/`robo-phrases` تبقى لاتينيّةً كما هي). الشاشة محفّزٌ فقط؛ النشاط الحقيقي يدويٌّ في الواقع. مبادئ صارمة:

1. **محلّي 100% دون إنترنت ودون ذكاء اصطناعي وقت التشغيل.** لا خادم/سحابة/نموذج لغوي/تتبّع. JSON/SVG محلي + WebAudio + مقاطع صوت مُضمّنة. (نماذج الصوت العصبية تُستعمل وقت التطوير فقط لتوليد MP3 ثابتة.)
2. **الآلي لا يُوصَف بـ«حيّ»** — اللفظ لخلق الله. البديل: متحرّك/متجاوب/نشيط. (ينطبق على نصوص الواجهة والتعليقات.)
3. **إطار إسلامي عقدي:** تدبّر خلق الله، علوم إسلامية، طب نبوي، علماء مسلمون، قيم بأدلّة. ردود الآليّ تنسب الفضل لله.
4. **خصوصية مطلقة + أمان عاطفي:** بيانات الطفل لا تغادر الجهاز؛ الآليّ هو من «يخطئ»، والطفل خبيرٌ واثق.

## البنية المعمارية

**vanilla JS** بلا إطار، عبر **Vite + vite-plugin-pwa**. كل صفحة `*.html` نقطةُ دخول مستقلّة (في `vite.config.js → rollupOptions.input`) تستورد من `src/` وبيانات من `content/`. `index.html` = **الحلقة الأساسية** (آلة حالات)؛ البقية أنشطة. الصفحات: index · home · basics · math · read · play · quiz · tales · puzzles · stories · progress · quran · lang · print · record. (وجذرُ المستودع `index.html` = الموقع الرسميّ الثابت، منفصلٌ عن تطبيق `/app/`.) **كلُّ صفحةٍ جديدةٍ تُضاف لـ`vite.config.js → rollupOptions.input`** وإلّا لم تُبنَ.

### الحلقة الأساسية (`index.html`)
آلة حالات: `idle → spark → task → teach(stage 0 جواب الطفل، stage 1 تعميق + «علّمني الخطأ») → heal`. كل ذكرى تُضيء جزءاً من الآليّ/عنصراً من عالمه؛ عند 8 مكافآت تعود ضحكته. تتضمّن: خريطة عالم (مناطق)، «فاجئني»، حقل جواب مكتوب، وضع الإخوة (coopMode)، و**لوحة التحكّم** (انظر قسم الحسابات أدناه). الانتقالات تُنطِق كلام الآليّ **وقت الانتقال** لا في كل إعادة رسم.

### الحسابات وحساب الأهل ولوحة التحكّم والتقارير
- `src/accounts.js` — حساباتٌ محلّيّة (localStorage `tilmithi_accounts_v1`؛ الجلسة في sessionStorage `tilmithi_current_user`). أدوار: **child** (ملفّاتُ أطفالٍ مُنشأة) · **guest** (مؤقّت) · **parent** (جلسةٌ تُبنى من إعدادات الوالد، ليست ملفّاً مخزَّناً) · وبلا-حساب (`default`). كلمةُ مرور الوالد (افتراضُها `admin`) + سؤالُ استعادة. **`loginParent(pw)`** يسجّل الوالدَ كجلسةٍ حقيقيّة (`isParent()`)، **`addProfile/updateProfile/deleteProfile`** لإدارة الأطفال.
- **دخولُ الوالد** من شاشة الدخول (`home.html`): كلمةُ المرور → حساب والدٍ يهبط على الفهرس فيستعرض المزايا بحسابه + بطاقة **«🛡️ لوحة التحكّم»**. **حماية:** أيُّ فتحٍ لـ`index.html?parent=1` = اللوحةُ وحدها (حاجزة، لا تُكشَف حلقةُ «علّم الآليّ» خلفها؛ `parentOnly` يُعتِم الـoverlay)؛ غيرُ الوالد يُطلَب كلمةَ المرور **كلَّ مرّة** (لا جلسةَ 5 دقائق)؛ الإغلاق ⇒ شاشة الدخول/الحساب الذي فُتحت منه. الوالدُ المسجَّل يفتحها مباشرةً (`openGate` يفحص `isParent()`).
- **لوحة التحكّم** (`openDashboard` في `index.html`): نظرة عامّة + **تقارير الأبناء** (`#sec-reports`: بطاقةٌ لكلّ طفلٍ — نجوم/ذكريات/أوسمة/أيّام نشطة + تفاصيل، تُقرأ بـ`getStatsFor(id)`/`getBadgesFor(id)`/`loadProgressFor(id)` من `progress.js`، قراءةٌ فقط) + التقدّم بالمجالات + إتقان لايتنر + مصادر الصوت + **إدارة الحسابات** (`#sec-accounts`: إضافة/تعديل/حذف طفلٍ، كلمة مرور) + كلمةُ مرور اللوحة + سؤالُ الاستعادة + إدارةُ تسجيلات التمرّن. **لوحةُ الوالد العُليا** في home (`setupView`): بطاقة «📦 المكوّنات» تكشفُ شبكةَ الأنشطة، وبطاقاتُ تقارير/حسابات/تحكّمٍ تَفتح اللوحةَ عند القسم المقصود (مرابط `#sec-…` + تمرير).

### نظام الصوت (الأكبر والأهمّ فهمه)
- `src/sound-prefs.js` — مفاتيح مستقلّة: `isTonesOn` (نبضة+مؤثرات)، `isVoiceOn` (نطق). و**اختيار مجموعة الصوت لكل نوع**: `getVoiceSet(type)`/`setVoiceSet(type,id)` حيث `type ∈ {letter, word, sentence}` والقيمة `"clips"` (espeak) أو مُعرّف مجموعة بشريّة.
- `src/sfx.js` (WebAudio، يحترم `isTonesOn`) + `src/robo-voice.js` (`roboBlip` نبضة + `roboSay`، يعيد تصدير العبارات من `robo-phrases.js`).
- `src/speak.js` — النطق: يجرّب `playClip` أولاً (مقطع مضمون) ثم Web Speech للنصوص الطويلة. يحترم `isVoiceOn`. يؤجّل أوّل نطق حتى `onvoiceschanged` ويختار صوتاً عربياً صراحةً.
- `src/tts-clips.js` — **محور التوجيه**: `classify(text)` (حرف/كلمة/جملة) ثم `srcFor` يختار المصدر بحسب المجموعة المحدّدة لنوعه: تسجيل بشريّ **مُدمج** (`voices-bundled.json`) أو **على الجهاز** (`voices.js`) أو مقطع espeak (`tts-manifest.json` / `tts/custom/`). يرجع لـespeak إن غاب. **النسخُ الوعديّة** (`playStoryAsync`/`playClipAsync`) موحَّدةٌ في **`playUrlAsync`** الذي **ينتظرُ جهوزيّةَ المقطع (`canplay`) قبل التشغيل** فلا يفشلُ أوّلُ تشغيلٍ لمقطعٍ لم يُحمَّل (كان يرتدّ خطأً للنطق الآليّ)؛ يحترمُ النموذجَ المختار (عصبيّ/بشريّ/آليّ) ولا يستبدله بسبب التحميل. **مبدأ:** يُحترَم نموذجُ الصوت المختار أيًّا كان.
- `src/voices.js` — تسجيلات بشريّة في IndexedDB (`tilmithi_voices`, v2 مع ترحيل): **كل مُسجِّل = مجموعة** (`setId = gender-age-name`، مفتاح `setId+SEP+text`). `primeVoices` يبني خريطة ذاكرة لكل المجموعات. `renameSet` يضمّ مجموعةً لأخرى.
- `src/voices-bundled.json` — المجموعات البشريّة المُدمجة (`{sets, files}`). تُبنى بـ`tools/gen-voices.mjs` من تصدير الاستوديو → `public/tts/voices/<setHash>/<textHash>.mp3`.
- `src/spell.js` — وضع التهجئة: `segment(text)` يحلّل الكلمة المشكّلة لمقاطع، `playSpelled` يشغّلها بالتتابع ثم الكلمة كاملةً (يستعمل `playClipAsync`).
- `record.html` — استوديو التسجيل: 3 تبويبات (حروف/كلمات/جمل) ووحدات، ملفّ مُسجِّل (اسم/جنس/فئة)، تصدير ZIP مصنّف (`src/zip.js`)، ضمّ مجموعات.
- `src/robo-companion.js` — رفيق الآليّ المُحقَن في صفحات الأنشطة (`cheer/encourage/recall/read/applaud`؛ تقبل `{then}` يُستدعى عند انتهاء الكلام). `speak`/`roboSay` تمرّران `onend` (مقطع→`onended`، نظام→`utterance.onend`) فلا تُقطَع ردّة الفعل.

التشغيل).** علامةُ `types` في المجموعات تُصفّي المنتقيات. **القرارُ النهائيّ (2026-06-19):** العصبيُّ Piper **ممتازٌ للجُمَل/النصوص/ردود الآلي، فاشلٌ للمفردات**؛ لذا kareem **`types:["sentence","story","reaction"]`** (`gen-piper` يولّد `SENTENCE_UNITS`+القصص فقط). **أنواعُ الصوت أربعةٌ** (`VOICE_TYPES`): حرف/كلمة/جملة/**reaction** (نوعٌ مستقلٌّ، `classify` يطابقُ بـ`REACTION_SET`=`REACTIONS`+`NOTICES`). **الافتراضُ لكلّ الأنواع = `AUTO` «أفضل المتاح»** (`sound-prefs.js`): يختارُ `srcAuto` في `tts-clips` لكلّ نصٍّ **بشريّاً (`deviceAnyURL` ثمّ المجموعاتُ المُدمجة غيرُ العصبيّة كـ«صالح») ← عصبيّاً (kareem) ← آليّاً (espeak)**. فالصوتُ البشريُّ (الأنضجُ) يُستعمَل حيثما سُجِّل ويتّسعُ تلقائيّاً كلّما غطّى الوالدُ المزيد، والعصبيُّ للجُمَل، والآليُّ تغطيةٌ كاملةٌ مضمونة. **يُمكن فرضُ مصدرٍ بعينه لكلّ نوعٍ في اللوحة** (espeak/صالح/عصبيّ متاحةٌ بحسب `types`؛ المنتقي يعرض «⭐ أفضل المتاح» أوّلاً). **espeak و«صالح» البشريّ (بلا `types`) متاحان لكلّ الأنواع.** (الأرقامُ مشكّلةٌ بالكامل في `numbers-ar.js`/`letters.js`/`numerals.js` لينطقَها espeak سليمةً؛ تاءُ التأنيث ة→ه وقفًا في `forSynthesis` بأمانٍ للمفردة المشكولة.) **قارئ القصص** (`src/story-reader.js`، في stories.html + tales.html): `attachStoryReader(host,{text,pages?,onPage?})` — زرّ تشغيل/إيقاف كبير + **قراءة تلقائيّة للقصّة كاملةً** (تنتقل صفحةً صفحةً) + **قائمة اختيار قارئ ظاهرة** (`readerOptions`/`effectiveReader`) + **مؤشّر مصدرٍ مرئيّ** (`storySourceKind` → 🧠 عصبيّ/🎙️ بشريّ/🤖 آليّ). يعتمد `playStoryAsync`/`stopClip`/`playClipForSet` (tts-clips) و`getStoryVoice`/`setStoryVoice`/`isStoryReadOn` (sound-prefs)؛ `storySet()` يحترمُ اختيارَ المستخدم (يشمل «آلي») وافتراضُه العصبيّ. **مزلقٌ مهمّ:** أزرارُ قارئ القصص محجوبةٌ كلّيّاً خلف توگل «قراءة ### تدفّق الاختبارات الموحّد
`src/quiz-flow.js` — `autoToggleHtml`/`wireAuto` (توگل «الانتقال التلقائيّ»، مفتاح `tilmithi_autonext_v1` في sound-prefs، افتراضُه يدويّ)، `onCorrect`/`onWrong` (هتاف/تشجيع ثمّ انتقالٌ بعد انتهاء الكلام أو زرٌّ يدويّ، وعند الخطأ زرُّ تجاوزٍ مع `recall`). مُطبَّق في `basics`/`math`/`quiz`. **`onCorrect(robo,view,next,praise,answer)`**: إن مُرِّر `answer` (حرف/رقم/كلمة/اسم شكل) **يَنطقه الآليّ أوّلًا ثمّ ردّةَ الفعل** ثمّ ينتقل (لا تتداخل الأصوات؛ `robo.read` يدعم `{then}`). في الألعاب: `celebrate()` بـ`play.html` تنطقُ الكلمةَ المبنيّة، و`quiz.html` ينطقُ الخيارَ الصحيح. **أمانُ التدفّق:** `speak.js → ttsPath` يُطلق `onend` بعد مهلةٍ إن لم تجهز أصواتُ المتصفّح (جهازٌ بلا أصواتٍ عربيّة) فلا يَعلَق الانتقالُ التلقائيّ. `src/confirm-action.js` أضاف `confirmChoice` (تأكيدٌ بزرّين دون كلمة مرور، يُستعمَل لتبديل الحساب وحذف تمرّنٍ ذاتيّ). **نمطُ «فكّرْ ثمّ أظهِرِ الخيارات» (كـGT‑SARARIM):** في الأسئلة ذاتِ الخيارات النصّيّة (`quiz` · `tales` أسئلةُ الفهم · `lang` اختبار · `math` الحساب) يُعرَض السؤالُ فقط، والخياراتُ مخفيّةٌ (`.opts.hidden`/`.grid.hidden`) حتى يضغطَ الطفلُ زرَّ **`.reveal`** «🤔 فكّرْ… ثمّ أظهِرِ الخيارات» — يُشجّعُ التذكّرَ قبل التعرّف. **أسئلةُ فهمِ القصص (`tales`) سؤالًا سؤالًا:** `showQ(i)` يَعرضُ سؤالًا واحدًا؛ بعد الإجابة يظهرُ زرُّ **`.nextq`** «السؤال التالي» الذي يكشفُ التالي (الأسئلةُ التاليةُ مخفيّةٌ حتى ذلك). **محتوى الأسئلة:** كلُّ قصّةٍ في `sararim-stories.js` لها 3–4 أسئلةٍ متنوّعة (حدثٌ + لحظة/تفصيل + العِبرة المستخلصة، 166 سؤالًا في الـ51 قصّة). **لا يُطبَّق على ألعاب الاختيار المباشر بالبطاقات** (حروف/أرقام/مطابقة). الأحجامُ الحاليّةُ للأزرار مناسبةٌ (طلبُ المستخدم: لا تُكبّرْها).

### الألعاب التنافسيّة والمطبوعات (`play.html` + `print.html`)
- **الألعابُ في `play.html`** قائمةُ بطاقاتٍ (`GAMES` → `open(id)`). ست منها **تنافسيّة** (`g.vs` → علامةُ `⚔️`): الذاكرة + دائرةٌ وعلامة/تحدّي الأعداد/صيدُ الحرف/سباقُ المطابقة/شبكةُ الكلمات. **الإطارُ المشترك:** `vs_opp`/`vs_ai` + `vsSetup(cfg)` (شاشةُ اختيار الصعوبة والخصم) + `vsMakePlayers(forceTwo)` (يبني لاعبَين بألوانٍ من solo/ai/الإخوة `p:<id>`/الوالد) + `vsReact(p0,p1)` (ردُّ النتيجة) + `vsScoreHtml`. الخصمُ AI بثلاث صعوبات (دائرةُ وعلامة: مينيماكس في الصعب؛ البقيّةُ احتماليّةُ نجاحٍ بحسب الصعوبة). **شبكةُ الكلمات** تتبُّعُ حروفِ الكلمة بالتسلسل (`wg_seq`) من شبكة 6×6 + تلوينُ خانات اللاعب.
- **صوتُ الألعاب:** `sfx` (لمسة/نجاح/خطأ) + **ردودُ الآليّ المنطوقةُ عصبيًّا**: نصوصُها في **`GAME_REACTIONS`** (`src/robo-phrases.js`، مشكولةٌ بلا إيموجي) → وحدةُ «games» في `vocab.js` → **مُولَّدةٌ بـ`gen:piper`** (kareem). تُعرَضُ وتُنطَقُ **بالنصّ نفسِه** عبر `robo.cheer/encourage/say(text)` (الصيغُ بلا نصٍّ تَنطقُ عبارةً عامّةً فتُخالفُ الفقاعة). **أيُّ تعديلٍ لنصوص الألعاب ⇒ أعِدْ `node tools/gen-piper.mjs`** وإلّا صَمَتَتْ على لينكس.
- **المطبوعات (`print.html`):** نماذجُ ورقيّةٌ جاهزة (`TEMPLATES`: دائرةٌ وعلامة بثلاثة أحجام · شبكةُ كلمات · صيدُ حرف · تحدّي أعداد · وصِّلْ بخطّ · ذاكرة · بنغو) + بطاقاتُ المكتبة + ملصق. **خاناتُ اختيارٍ لكلّ لعبةٍ** (`#tplChecks`/`tAll`) + **عددُ النُّسخ (`#copies` 1–4)** — كلُّ نسخةٍ تُولَّدُ بمحتوًى مختلفٍ (الدوالُّ تُعشِّي عشوائيًّا). بطاقةُ «print» في `play.html` تفتحُ `print.html` (لا التعليماتِ الداخليّة).
- **الطباعةُ على المنصّات (`src/smart-print.js` → `smartPrint(el,filename)`):** على الويب/سطح المكتب `window.print()` (وفي `print.html` الـ`.tpl` بـ`break-inside:avoid` فتتراصُّ دون قطع). على **الهاتف (Capacitor)** `window.print` لا يعمل: نُنقّطُ بـ**`html2canvas`** (تشكيلُ العربيّةِ صحيحٌ — `pdf.html` يُشوّهُها) بعرضِ A4 ثابت (794px) ثمّ **نَحشُّ كتلَ `.tpl` في الصفحات دون قطعِ لعبة**، ونكتبُ ونشاركُ عبر **`@capacitor/filesystem`+`@capacitor/share`** (تُستدعى من `window.Capacitor.Plugins`).

### السمة (الوضع الداكن) والتجاوب
- **`src/theme.js`** — تبديلُ السمة فاتح/داكن (localStorage `tilmithi_theme`؛ يحترمُ `prefers-color-scheme` عند عدم الاختيار). `wireThemeToggle(btn)` يربط زرَّ 🌙/☀️ (في ترويسة `home.html`، يتحكّم بالكلّ). يستوردُ `src/dark.css` و`src/topbar.css` و`src/nav-icons.css`. **مسنّنُ لوحة التحكّم (`mountPanelGear`):** يُحقَن تلقائيّاً (أثرٌ جانبيٌّ عند الاستيراد، theme.js مُستورَدٌ في كلّ الصفحات عدا print) **ملاصقًا لزرّ تبديل السمة** (`#themeBtn` أو `.iconbtn[title*="الوضع"]`) في شريط كلِّ صفحة (`#panelGear` → `index.html?parent=1`، حاجزُ كلمة المرور يحميه) فيَصِلُ الوالدُ للّوحة مباشرةً دون العودة للفهرس؛ لا يُحقَن في `index.html` (مُضيفُ اللوحة). أيقونتُه **ترسُ Font Awesome** (`iconHtml("gear")` — نفسُ سلسلة أيقونات الفهرس؛ أُضيف `gear` في `gen-fa-icons.mjs`، أعِدْ `npm run gen:icons` عند تغييره) بلون القسم ودورانٍ عند المرور. وزرُّ السمة يأخذ صنفَ `.themetoggle` لنفس تأثير الدوران.
- **أزرارُ الرجوع خطوةً للوراء** موحّدةٌ بصنف **`.backstep`** (في `topbar.css` العامّ): حبّةٌ بحدٍّ بلون قسمها (`--primary`) وسهمِ رجوعٍ `↩` (عبر `::before`) تمتلئ عند اللمس — واضحةٌ بديهيّةٌ للطفل، **مستقلّةٌ عن `.back`** (المستعمَل لأزرارٍ ثانويّةٍ أخرى كـ🔊). مطبَّقةٌ في `backBar()` (basics/math/play/read) وقصص/حكايات/قرآن وأزرار math الفرعيّة؛ ولانغ صنفُه `.backbtn` بالمظهر نفسِه (تسميتُه تحوي `↩` فلا `::before`). لا تُغيّرْ زرَّ الفهرس (`.topbtn`).
- **`src/dark.css`** — السمةُ الداكنة المشتركة (`html[data-theme="dark"]{…}`): تُجاوِزُ متغيّراتِ الألوان الموحّدة + الأصنافَ الشائعة + خصوصيّاتِ الفهرس/شاشة الدخول. تُستورَد في وحدة كلِّ صفحة (`import "./src/dark.css"`) فيستخرجها Vite كـ`<link>` في الإنتاج (**بلا وميض**). **مفعَّلةٌ في كلّ الصفحات عدا `print`** (الطباعة تبقى فاتحة).
- **منعُ الوميض:** كلُّ صفحةٍ بها سكربتٌ كلاسيكيٌّ صغيرٌ في `<head>` يضبط `data-theme` قبل الرسم. الإضافةُ معمَّمةٌ عبر سكربت python (انظر سجلّ الالتزامات) — السطرُ نفسُه في كلّ الصفحات.
- **مزلق:** الألوانُ المثبّتةُ يدويّاً بأنماطٍ **inline** لا تتبع المتغيّرات؛ تُعالَج بانتقاءات `dark.css` (أصناف) أو `!important` للـinline. عند إضافة سطحٍ جديدٍ استعمل `var(--card)`/`var(--bg)`/`var(--ink)` ليتبع السمةَ تلقائياً.
- **شموليّة dark.css:** يُغطّي الأسطحَ الكريميّة (`.tad/.picview/.objbox/.domhead/.note/.hintbox/.tab/.lvl…`) والتغذيةَ الراجعة (صحيح أخضر/خطأ أحمر) وعناصرَ النماذج (`!important` ليتغلّب على `background:#fff` المضمَّن) وورقَ المصحف (`.mushaf` داكن، نصّ فاتح) ومُسجّلَ القراءة (`.rr`) وبطاقاتِ الفهرس. كثيرٌ من `background:#fff` في الصفحات حُوِّل إلى `var(--card)` (لا يُغيّر الفاتح، يَدكَن في الداكن). زرُّ 🌙/☀️ في كلّ صفحة.
- **التجاوب وتصميم الفهرس:** بطاقاتُ الفهرس مدمجةٌ مربّعةٌ ملوّنة (أيقونة+عنوان، متحرّكة عند التمرير/النقر؛ نمط `.navcard`، لوحة ألوان nth-child) للطفل/الضيف؛ والبطاقةُ الكلاسيكيّةُ بالوصف (`.nav.detailed`) في قسم الأهل فقط (لوحة الوالد العُليا). الترتيب: علّم الآليّ→القرآن→الأساسيّات→سُلّم القراءة→الرياضيّات→اللغات→القصص→الألعاب→الألغاز→الاختبارات→تقدّمي، وأدواتُ الأهل أسفلَه. تجاوبُ صفوف لوحة التحكّم على الهاتف عبر `@media(max-width:560px)` في `index.html`.
- **لوحة التحكّم (مطويات + إغلاق تلقائيّ):** كلُّ قسمٍ (`.sec`) بطاقةٌ مطويّةٌ افتراضاً (العنوان يفتح/يطوي؛ القسمُ المقصودُ بالرابط `#sec-reports`/`#sec-accounts` يُفتَح ويُمرَّر إليه)؛ وتُغلَق اللوحةُ تلقائيّاً بعد **30 ثانية خمول** (`armIdle`؛ أيّ نقر/كتابة/تمرير يعيد العدّ).

### القرآن الكريم (`quran.html` + `content/quran.js`)
الفاتحة + جزء عمّ (آخر حزبين، السور 78–114؛ 38 سورة/571 آية). `content/quran.js` = النصّ العثمانيّ (`{surahs:[{n,name,type,ayahCount,ayahs:[{a,text}]}]}`). **التلاوة:** الحصري آيةً آية في `public/quran/husary/SSSAAA.mp3` (محلّيّة). **السور مرتّبةٌ من الأقصر إلى الأطول** (تدرّجٌ في الحفظ) + تصفيةٌ بالمستوى (قصيرة ≤7/متوسّطة ≤25) افتراضُها بحسب `getChildAge`. عرضٌ آيةً آية بخطوطٍ قرآنيّة (Amiri Quran / Colored / Uthmanic Hafs في `public/fonts/`، اختيارٌ محفوظ)، **تظليلُ الآية المقروءة**، «اقرأ السورة» (تتابعٌ تلقائيّ) + «كرّر الآية» + «كرّر السورة» للحفظ. **الترتيب:** «الأقصر أولًا» (تدرّجٌ، الافتراض) أو «قرآنيّ» — والفاتحةُ دائمًا أوّلًا. لكلِّ آيةٍ **تسجيلُ تلاوة** (عبر `attachReader` مع `opts.onHelp`=تلاوة الحصري المرجعيّة + `helpLabel`). **البسملة** (لغير الفاتحة، كمنطق GT-QURANREADER): تلاوةُ الحصري في everyayah **لا تتضمّن البسملة في ملفّ الآية الأولى** (تأكَّدَ بالمدّة: العصر آية1 «والعصر» = 2.7ث، والبسملة وحدها 5ث). فالبسملةُ **صفٌّ مستقلّ** نصًّا وصوتًا: صوتُها = ملفّ الفاتحة آية1 (`BASMALA_AUDIO = 001001.mp3`)، والآيةُ الأولى تُعرَض وتُتلى **بلا بسملة** (`verseNoBism` يفصلها بمطابقةٍ غيرِ حسّاسةٍ للتشكيل — لا تَعتمِدْ على `startsWith` لثابتٍ نصّيّ). «اقرأ السورة» يبدأ بالبسملة (001001) ثمّ الآيات؛ النقرُ على صفّ البسملة يشغّلها وحدها؛ توگل يُظهرها/يُخفيها (فتُكرَّر الآيةُ الأولى بلا بسملةٍ للحفظ). **أرقام الآيات بالأرقام الغربيّة** كبقيّة الواجهة (بطلب المستخدم). **التقدّم:** `quran_open`/`quran_listen` + قسم «القرآن» + وسامان (صديقُ القرآن/يَحفظ) + مهمّةٌ يوميّة. **مزلق الحجم:** تلاوة الحصري ~79م لا تُحمَّل مسبقاً (`globIgnores`) بل **CacheFirst وقت التشغيل** (`runtimeCaching` → cacheName `quran-husary`) فتعمل دون إنترنت بعد أوّل استماع؛ النصّ مُضمَّنٌ فيعملُ دائمًا.

### اللغات الأجنبيّة (`lang.html` + `content/lang-en.js` / `lang-fr.js`)
قسمٌ مستقلّ: بطاقة الفهرس «🌍 لغات أجنبيّة» → `lang.html` (اختيار en/fr ثمّ أقسامٌ بمعاملات URL `?lng=&sec=&cat=&story=&pg=`). **أقسامٌ (`SECTIONS`):** الأحرف · الأرقام (0–100) · الكلمات (18 فئةً مواضيعيّة، كلمة+معنى عربيّ+رمز) · **عبارات وجمل** (فئاتٌ منها «كيف أسأل؟» الأسئلةُ اليوميّة وجُمَلٌ مفيدةٌ أطول) · قصص مصوّرة (وفيها قصّةُ «رحلة الأرقام» برسومٍ أصليّة وعباراتٍ داخل الصور حسب اللغة عبر حقل `caption`) · **الأفعال والأزمنة** · اختبار · ألعاب. `content/lang-<code>.js` يصدّر `{code, name, nameAr, flag, dir, voice, letters, numbers, words, phrases, verbs, stories}`. **قسمُ «ألعاب» مركزٌ** (`?sec=games&g=match|matchar|spell|listen`): مطابقةُ الصور · مطابقةُ المعنى (لغة↔عربيّ) · الإملاء/التهجئة · استماعٌ واختيار. **قسمُ «الأفعال والأزمنة»** (`renderVerbs`): لكلّ فعلٍ تصريفٌ في المضارع لكلّ الضمائر (`conj[]`) + الأزمنةُ الثلاثُ لصيغة المتكلّم (`tenses`) + زرُّ تسجيلِ نُطق الطفل. (الشخصيّةُ الودودةُ تُسمّى «الآليّ».) **التنقّلُ ببطاقاتٍ هرميّةٍ ملوّنةٍ متحرّكة** (`.lcard`): لغة→أقسام→(فئات)→محتوى، مع زرّ رجوعٍ `.backbtn[data-go]` (`langs/secs/cats/stlist`). **عنوانُ كلّ بطاقةِ قسمٍ/فئةٍ يتبدّل بين العربيّة ولغة القسم كلَّ 5ث** (تعزيزٌ للتعلّم؛ `startTitleAlt` يبدّل `.lt[data-fg]` بتلاشٍ، ويُلغى ويُعاد ضبطُه عند كلّ تنقّل؛ ترجماتُ الأقسام في `SECTIONS[k]` والفئاتِ في `CAT_I18N`). **نطقُ الحروف (عصبيٌّ كامل):** قسمُ اللغات يُنطَقُ **بالعصبيّ Piper كاملاً** (أحرف+كلمات+عبارات+جمل+نصوص) — قرارُ المستخدم. كلُّ حرفٍ له حقلُ `name` = اسمُه صوتيّاً ككلمةٍ (H→`aitch`، الفرنسيّة H→`ache`) يُستعمَل **مفتاحاً** ونصَّ تركيب. **مزلقُ الإملاء الصوتيّ:** المفتاحُ هو ما يُغذّي مُفنِّمَ espeak داخلَ Piper، فإن كان الإملاءُ خاطئاً اختلَّ النطقُ — مثالٌ مُصلَح (2026-06-20): A الإنجليزيّة كانت `name:"ay"` فيُفنِّمُها espeak `/aɪ/` = نطقَ **I**! صُحِّحت إلى `name:"eigh"` (`/eɪ/`). **عند شكوى نطقِ حرفٍ تحقّقْ أوّلاً من فونيماته:** `espeak-ng -v en-gb -q --ipa -- "<name>"` (أو `-v fr`) وقارِنْه باسم الحرف القياسيّ. **تثبيتُ المقاطع القصيرة:** اسمُ الحرف وحدةٌ قصيرةٌ جدّاً يُشوّهُ النموذجُ العصبيُّ حوافَّها؛ لذا `gen-piper-lang` **يُذيّلُ نصَّ تركيبِ أسماء الأحرف بنقطة** (`name + " ."` خاتمةٌ جُمليّةٌ تُحسّنُ النطق) مع إبقاء المفتاح = `name`. **لا تستبدلِ العصبيَّ بـespeak في اللغات** (جُرِّب فرُفِض): الإصلاحُ يكون بضبط الإملاء/التثبيت لا بتغيير المحرّك. **القصص = ترجمةُ `content/stories.js`** بنفس `id`/`cover`/`art` (تُعاد رسومُها من `src/story-art.js`) — فأيُّ قصّةٍ عربيّةٍ جديدةٍ تُضاف ترجمتُها هنا بنفس المفاتيح. قارئُ القصّة الكاملة بزرّ «📖 القصّة كاملةً» (تتابعٌ مع تقليب + إيقاف). **النطق:** عصبيٌّ Piper مُدمَجٌ offline — `en_GB-alba`→مجموعة `tts-alba`، `fr_FR-tom`→`tts-tom` (تُولَّد بـ`tools/gen-piper-lang.mjs en|fr`؛ النصوصُ = أسماءُ الحروف + الكلمات + الأرقام + **العبارات** + **الأفعال (المصدر+التصريف+الأزمنة)** + القصص؛ النماذج `.onnx*` مُستثناةٌ من git، المقاطعُ تُرفَع). **عند تغيير محتوى لغةٍ يُعاد توليدُ مجموعتها** ثمّ تُنظَّف المقاطعُ اليتيمة (غيرُ المُشار إليها في `voices-bundled.json`) من مجلّد المجموعة. `lang.html` ينطق عبر `playClipForSet(text, L.voice)` و`playClipForSetAsync` (للقراءة المتتابعة)، وإلّا Web Speech بلغة القسم. المجموعتان `types:["foreign"]` فلا تظهران في قوائم الصوت العربيّة. نصوصُ اللغة LTR ضمن واجهةٍ RTL (صنف `.ltr`). `piper_synth.py` يقبل معامِلَ متحدّثٍ رابعاً (للنماذج متعدّدة الأصوات).

## مزالق حرجة

- **التخزين مرتبطٌ بالأصل (المنفذ):** الحسابات/التقدّم (localStorage) والتسجيلات (IndexedDB) مرتبطةٌ بـ`http://localhost:<port>`؛ فتغيّرُ منفذِ Vite (5173↔5174) يُخفي بياناتِ التطوير (لا تُفقَد، بل على أصلٍ آخر). لذا `vite.config.js` يثبّت `server/preview.port=5174 + strictPort`.

- **عامل الخدمة (PWA) يُخبّئ كلَّ شيء:** بعد إعادة البناء قد يُقدّم كوداً/مقاطعَ قديمة. الحلول المُطبَّقة: في الإنتاج `workbox.skipWaiting+clientsClaim+cleanupOutdatedCaches` (تحديثٌ فوريّ)، وفي التطوير حارسٌ في `src/sound-prefs.js` يُلغي تلقائياً أيَّ SW قديمٍ ويمسحُ الكاش عند `import.meta.env.DEV` (فلا يُقدَّم كودٌ مُخبّأ أثناء `npm run dev`). إن ظهر سلوكٌ قديمٌ رغم ذلك → أعِد التحميل مرّتين أو ألغِ تسجيلَ SW يدويّاً.
- **مفتاح تقدّمٍ لكلّ حساب:** التقدّمُ مفصولٌ لكلّ ملفّ — `index.html` و`src/progress.js` يكتبان في **`localStorage["tilmithi_progress_v1__<id>"]`** للملفّ النشط غير الضيف (وإلّا المفتاح الأساسيّ للضيف/بلا-حساب). **المفتاح يُحسَب مرّةً عند استيراد الوحدة** (من `getCurrentUser()`)؛ فلقراءة تقدّمِ طفلٍ آخرَ (التقارير) استعمل `getStatsFor(id)`/`loadProgressFor(id)` (قراءةٌ مباشرةٌ من مفتاحه). داخل المفتاح الواحد: الملفّان يتشاركان الكائن بحقولٍ مختلفة → استعمل دائماً نمط **حمّل-عدّل-احفظ**، لا تكتب كائناً كاملاً من ذاكرةٍ قديمة.
- **`classList.add("")` يرمي استثناءً** — احرس ضدّ القيمة الفارغة (سبب تعطّل أزرار القراءة سابقاً).
- **مطابقة المقاطع حرفية:** `playClip(text)`/التسجيلات تطابق النصّ تماماً. النصوص المركّبة وقت التشغيل ترجع لـWeb Speech. بعد إضافة محتوى منطوق: `npm run gen:audio`.
- **ترتيب التشكيل (`src/arabic-normalize.js`):** espeak والعصبيُّ Piper يتوقّعان «حرف + شدّة + حركة»؛ ترتيبُ «حرف + حركة + شدّة» (وما يُنتجه NFC) يُفقِد نطقَ الشدّة فتُسمَع كالسكون (أوّل→أَوْل، محرّم، عدّ، جنّة)، ولفظُ الجلالة المشكَّل يُشوَّه بينما المجرَّد «الله» يُنطَق صحيحاً. الحلّ: **`forSynthesis(text)`** (إعادةُ ترتيب الشدّة قبل الحركة + تجريدُ لفظ الجلالة) يُطبَّق على **نصّ التركيب فقط** في `gen-audio`/`gen-piper` (المفتاحُ يبقى الأصل، فلا يتغيّر اللوكب)، و**`reorderMarks`** في `spell.js segment()` ليطابق التقطيعُ مفاتيحَ المقاطع (الاستوديو + المولَّدة، التي تستعمل ترتيبَ syl.js «شدّة+حركة»). أيُّ تغييرٍ في نطق الشدّة/الجلالة ⇒ أعِدْ `gen:audio` + `gen:piper`.
- **حدود espeak (مقاطع آليّة):** السكون على ب/ج/د/ض والهمزة أْ = صمت؛ نِ تلتبس بـذِ؛ الشدّة تقريبية. تُحلّ بتسجيلٍ بشريّ (الاستوديو/`public/tts/custom/`). موثّقة في `src/tts-clips.js`/CREDITS.
- **espeak كمحلّل صوتيّ عصبيّ (Piper):** النماذج العصبية طبيعيّةٌ للجمل، ضعيفةٌ للحروف المنفردة وتطمس الشكل → غير مناسبةٍ لجوهر التعليم؛ تصلح للكلمات/الجمل فقط (عبر اختيار النوع).
- **Node-safety:** ما يستورده `gen-audio`/`gen-voices` (Node) يجب أن يخلو من `window` — لذلك `vocab.js`/`syl.js`/`robo-phrases.js`/`islamic.js` بلا DOM (`robo-voice.js`/`speak.js` ليست كذلك).
